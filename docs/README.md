# PartyPad documentation

Long-lived design notes and implementation roadmaps live here so project
direction is not dependent on issue trackers or prior agent conversations.

## Roadmaps

- [`roadmaps/umd-self-service.md`](roadmaps/umd-self-service.md): turn the
  current owner-operated online prototype into a public self-service,
  cross-platform release.

## Protocols

- [`device-authorization-protocol.md`](device-authorization-protocol.md):
  versioned device-code authorization, revocable desktop credentials, session
  authentication, durable schema, and data retention.

## Service policies and operations

- [`privacy.md`](privacy.md): pre-pilot data and retention summary.
- [`acceptable-use.md`](acceptable-use.md): pre-pilot abuse expectations.
- [`operations.md`](operations.md): deployment, kill switch, metrics, billing,
  and rollback runbook.
- [`signing-and-release.md`](signing-and-release.md): unsigned alpha workflow
  and the protected Windows/macOS signing gates.

Roadmaps describe intended work, not current guarantees. The root
[`README.md`](../README.md) remains the source of truth for behavior users can
run today, and [`AGENTS.md`](../AGENTS.md) contains mandatory agent constraints.
